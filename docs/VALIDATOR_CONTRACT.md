# Контракт валидатора задачи

Валидатор — это **один файл** `validator.py` в корне рабочей копии репозитория песочницы (тот же каталог, что и для команд Git).

## Поведение

- Платформа записывает в `validator.py` содержимое ассета типа `VALIDATOR` из БД (если ассет задан и не пустой).
- **Локальная песочница**: выполняется `sys.executable validator.py` с `cwd`, равным корню репозитория. Код возврата `0` — успех, любой другой — провал.
- **Docker-песочница**: выполняется `docker exec <container> python3|python validator.py` без оболочки; рабочий каталог контейнера — `/workspace` (смонтированный том с репозиторием). Контейнер поднимается с `--network none`, CPU/RAM/pids ограничены (см. `SANDBOX_DOCKER_*` в `apps/core/services/sandbox_ops.py`).
- Таймаут — `session.timeout_seconds` (см. модель `SandboxSession`).
- При таймауте или необработанной ошибке запись попытки получает вердикт `ERROR`, иначе `PASSED` / `FAILED` по коду возврата.

## Ограничения для авторов задач

- Валидатор не должен рассчитывать на сеть (в Docker она отключена).
- Не полагайтесь на произвольные пути вне корня репозитория: проверки должны использовать только файлы внутри текущего каталога.
